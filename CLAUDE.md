# CDP Generation — главные инструкции для Claude Code

> Этот файл автоматически загружается в каждую сессию Claude Code в этом проекте. Здесь — обязательные правила работы. Изменения в этом файле = изменения поведения ассистента.

## 🎯 О проекте в одну фразу

**CDP Generation** — AI-инструмент для автоматической генерации индивидуальных коммерческих предложений по продуктам Calltouch CDP под конкретных клиентов. Получает на вход URL сайта клиента и бриф, выдаёт готовый PDF со сценариями.

> Полная концепция: см. [`Project_IDEA.md`](./Project_IDEA.md)
> Спецификация: см. [`docs/spec.md`](./docs/spec.md)

## 🛠️ Технологический стек

- **Язык:** Python 3.12
- **LLM:** Anthropic-only — Claude Opus 4.7 (творческое ядро) + Claude Haiku 4.5 (дешёвый extract). Изменено 2026-05-07: Gemini/Groq убраны из фазы 1 ради простоты setup'a, см. `docs/spec.md` §5.
- **Веб-интерфейс:** Streamlit (фаза 1, локально) → FastAPI + Next.js (фаза 2, облако)
- **БД:** SQLite (фаза 1, локально) → **Supabase** (фаза 2, облако)
- **Auth:** None (фаза 1, локально) → **Telegram Login Widget** (фаза 2)
- **Deploy:** локально → **GitHub** (CI/CD) → Streamlit Cloud / Vercel / Render
- **MCP-серверы:** Context7 (документация), Supabase (БД)
- **Брендинг:** фиолетовый Calltouch + морской cyan (см. [`docs/visual_identity.md`](./docs/visual_identity.md))

## ⚖️ ОБЯЗАТЕЛЬНЫЕ правила работы (НЕ нарушать)

### 1. Context7 — единственный источник правды по библиотекам и API

Перед использованием **любой** внешней библиотеки, SDK, API или CLI — **обязательно** обратиться в Context7 MCP за актуальной документацией:

- `mcp__claude_ai_Context7__resolve-library-id` — найти библиотеку
- `mcp__claude_ai_Context7__query-docs` — получить документацию

Это касается: anthropic SDK, streamlit, fastapi, supabase-py, telegram-login, pydantic, beautifulsoup, jinja2, weasyprint, vercel, render и любых других. Запрещено писать код по памяти, не свериваясь с актуальной документацией Context7.

### 2. Supabase — только через Supabase MCP

Любые операции с базой данных (создание таблиц, миграции, RLS-политики, queries) — через MCP-инструменты Supabase:

- `mcp__claude_ai_Supabase__list_projects`, `list_tables`, `apply_migration`, `execute_sql`, `get_advisors`, `generate_typescript_types` и т.д.

Перед `apply_migration` всегда проверять схему через `list_tables`. Перед коммитом миграции — прогонять `get_advisors` (security + performance).

### 3. GitHub — через `gh` CLI и `git`

Версионирование, ветки, PR, релизы, деплой — через GitHub. Локальные изменения коммитятся в feature-ветки, в `main` мержатся через PR. Деплой — через GitHub Actions.

- `git status`, `git diff`, `git log`, `git checkout -b`, `git commit`, `git push`
- `gh pr create`, `gh pr view`, `gh issue create`, `gh release create`

Никаких force-push в `main`. Никаких коммитов прямых в `main` без PR. Никаких `--no-verify` на коммитах без явного указания пользователя.

### 4. Не упоминать конкурентов Calltouch без явного запроса

В коде, документации, промптах и в выходных PDF — не вводить по своей инициативе названия Mindbox, Altcraft, enKod, Sendsay, Carrotquest, Roistat, Rees46. Если клиент или менеджер сам поднял тему — обсуждать нормально.

### 5. Актуальные цены продуктов Calltouch

- **CDP Calltouch:** от 10 500 ₽/мес. Скоринговая модель и триггер «Выявлен интерес» включены в базовый тариф
- **Big Data партнёров:** от 5 000 ₽/мес
- **WABA / WhatsApp Business:** не вставлять по умолчанию в КП — только если клиент или менеджер явно поднял тему

### 6. Стиль кода

- Тип-аннотации обязательны (Python 3.12 syntax: `list[str]`, `int | None`)
- Pydantic v2 для всех контрактов данных
- Без избыточных абстракций — три строки лучше преждевременного класса
- Без комментариев, описывающих *что* делает код (имена идентификаторов уже это рассказывают). Комментарий уместен только если объясняет *почему* — нетривиальное решение, обход бага, скрытое ограничение
- Без `try/except Exception` без понятной обработки

## 🤖 Субагенты

В `.claude/agents/` определены 9 специализированных субагентов. Используй их **проактивно**, не дожидаясь явного запроса от пользователя:

| Субагент | Когда вызывать |
|---|---|
| `architect` | Проектирование архитектуры, разбивка задач на этапы, выбор подхода |
| `prompt-engineer` | Любая работа с промптами для LLM, оптимизация качества, A/B-тесты |
| `scenario-author` | Генерация сценариев CDP под конкретного клиента |
| `visual-designer` | HTML/CSS, брендинг, PDF-шаблоны |
| `data-scout` | Парсинг сайта клиента, извлечение фактов, RAG |
| `reviewer` | Финальный ревью КП на соответствие правилам Calltouch |
| `db-architect` | Любая работа с Supabase, схемы, миграции, RLS |
| `deploy-engineer` | GitHub, CI/CD, деплой, инфраструктура |
| `doc-keeper` | Перед написанием кода с любой библиотекой — обращение в Context7 |

Подробности каждого субагента — в файлах `.claude/agents/<name>.md`.

## 🔁 Фазы разработки (двухфазная архитектура)

- **Фаза 1 (текущая):** локальный Python + Streamlit, SQLite, без авторизации, тестирование на машине пользователя
- **Фаза 2 (после стабилизации):** Supabase для БД, Telegram Login Widget для входа, деплой на облачный хостинг через GitHub

Не начинать работы фазы 2 пока фаза 1 не зафиксирована (рабочий MVP, который пользователь использует руками).
