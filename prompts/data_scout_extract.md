# Что нужно вытащить (поля схемы SiteFacts)

- `primary_domain` — главный домен, как дан выше.
- `additional_domains` — дополнительные домены, если упомянуты.
- `company_name` — официальное название компании (бренд, юр. название).
- `industry_detected` — отрасль одной фразой на русском (например: «премиум-домостроение», «застройщик жилья», «онлайн-школа»).
- `detected_class` — один из: `mass`, `comfort`, `business`, `premium`, `deluxe` (или null, если непонятно).
- `products` — ключевые продукты/услуги. Для каждого: name, url (если есть), price_from (число в рублях, если есть) и краткое description (1 фраза).
- `projects` — реализованные проекты/кейсы. name, url, location, price_from, status.
- `pricing_signals` — фрагменты ценовой политики: «средний чек 25 млн», «от 65 000 ₽/м³». В виде {{label, value}}.
- `team_members` — публичные имена с ролями: владелец, директор, ключевые архитекторы.
- `contact_blocks` — контакты: kind ∈ {{phone, email, address, office, messenger}}, value, optional note.
- `social_channels` — соцсети: platform ∈ {{youtube, telegram, vk, max, instagram, facebook, tiktok, rutube, other}}, handle, url, subscribers (если упомянуто).
- `detected_crm_hints` — намёки на CRM/системы (Bitrix24, amoCRM, retailCRM и т.п.).
- `raw_summary` — 3–5 предложений на русском о бизнесе клиента, чтобы менеджер сразу узнал свою компанию.

# Жёсткие требования
- Не выдумывай факты, которых нет в содержимом. Если поле неизвестно — null или пустой список.
- Цены — только если они явно указаны на сайте. Никаких «обычно стоит».
- Имена — только из явных страниц «о команде», «о компании», подписей, цитат.
- `raw_summary` — это сухая фактологическая выжимка, без маркетинга и без «мы рекомендуем».

Верни СТРОГО валидный JSON по схеме `SiteFacts`. Никаких пояснений вне JSON.
