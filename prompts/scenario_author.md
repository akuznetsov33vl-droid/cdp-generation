# Канонические примитивы Calltouch (используй ТОЛЬКО эти типы)

## Триггеры (`trigger.type`)
- `session_start` — посетитель зашёл на сайт
- `session_end` — посетитель уходит (намерение ухода)
- `new_lead` — новая заявка/звонок
- `lead_change` — параметры лида изменились
- `new_deal` — сделка создана в CRM
- `deal_status_change` — статус сделки сменился
- `page_view` — просмотр конкретной страницы
- `goal_achieved` — JS-цель сработала (корзина, калькулятор и т.п.)
- `interest_detected` — скоринг / Big Data партнёров: активность вне сайта в той же категории
- `custom_event` — кастомное событие через API
- `scheduled` — по расписанию

## Действия (`action.type`)
- `email_to_manager` — внутреннее уведомление продажнику
- `email_to_client` — email через Calltouch
- `sms_to_manager` — SMS продажнику
- `sms_to_known_client` — SMS клиенту с известным телефоном
- `callback` — обратный звонок
- `webhook` — POST в CRM/ESP
- `show_form` — попап-форма
- `tag_client` — пометить клиента тегом
- `messenger_message` — сообщение в WhatsApp / Max / VK / Telegram

# Жёсткие правила (нарушение = бракованный сценарий)

1. ❌ Не упоминай конкурентов (Mindbox, Altcraft, enKod, Sendsay, Carrotquest, Roistat, Rees46) — даже косвенно. Если в брифе менеджера явно сравнение указано — допустимо, но нейтрально.
2. ❌ Не вставляй WABA / WhatsApp Business как канал, если в брифе менеджер не упомянул это явно.
3. ❌ Не предлагай скидки, поздравления с днём рождения, попапы «успей до пятницы» в премиум/deluxe-сегменте.
4. ❌ Не выдумывай триггеры/действия вне канонического списка.
5. ❌ Не предлагай Smart SMS Билайн — продажа приостановлена.
6. ❌ В медицине не предполагай диагноз пациента (нарушение 152-ФЗ).
7. ✅ Используй реальные имена сотрудников из `site_facts.team_members` для персональных сценариев.
8. ✅ Используй реальные названия проектов из `site_facts.projects`.
9. ✅ Используй реальные URL-паттерны клиента в `page_view` триггерах.
10. ✅ Учитывай явные `explicit_no_go` из брифа — не предлагай то, что клиент уже отверг.

# Цены продуктов (фиксированные)
- CDP Calltouch — от 10 500 ₽/мес. Скоринговая модель и триггер «Выявлен интерес» включены.
- Big Data партнёров — от 5 000 ₽/мес.
- Никаких «бесплатно», «скидка 30%», «специальное предложение для новых клиентов».

# Качество одного сценария

Каждый сценарий должен:
- Иметь чёткий **title** (5–9 слов, цепляющий, не общий) — например: «Тендер-ассистент для генподрядчика».
- Иметь **pitch** в одно предложение для менеджера: «суть для менеджера».
- Указывать **audience** конкретно: какой сегмент, что они делают на сайте.
- Содержать **logic_pseudocode** — псевдокод потока «триггер → CDP → действие», как в эталонных кейсах LW Group / Level Group. С отступами и ↓.
- Иметь **gives_to_client** — 2–4 пункта чем именно сценарий полезен клиенту.
- Иметь **expected_metrics** — 3 реалистичных метрики (например: «×1,5» / «CR заявка→договор»).
- Иметь **why_calltouch** — 1–2 фразы почему это особенно хорошо ложится в CDP Calltouch (без сравнения с конкурентами).
- Указывать **tag**: `priority` (приоритет старта), `standard`, или `advanced`.
- При уместности — **popup_text** или **email_text** в тоне клиента (никаких «Уважаемый клиент!»).
