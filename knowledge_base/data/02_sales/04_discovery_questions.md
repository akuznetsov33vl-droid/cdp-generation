# 4. Вопросы для выявления потребностей

## Принципы выявления

### Главный принцип

> **Не нужно предлагать клиентам всё подряд** — необходимо услышать его задачи и подобрать оптимальный продукт.

### Метод ВСПВ

| Буква | Значение | Цель |
|-------|----------|------|
| **В** | Возможность | Как сейчас |
| **С** | Ситуация | Что не работает |
| **П** | Проблема | Болевая точка |
| **В** | Выгода | Что получит |

### Открытые вопросы для начала

- Как вообще дела? Какие есть сложности?
- Слышали про наши новые продукты?
- С какой аудиторией работаете в первую очередь: новой или текущей?
- Как дела с клиентами? KPI?
- Подскажите, как вы привлекаете новую аудиторию клиентов?
- Как вы работаете с текущей базой клиентов?

**Логика:** если всё ок — продукт, чтобы сделать ещё лучше. Если не ок — продукт, чтобы стало ок.

## Анализ перед звонком (что смотреть в проекте)

### 1. Какие источники отслеживаются

- Много сделок без источника → выход на Big Data
- Неотслеживаемый трафик → предлог для звонка
- Ключевые каналы → дожать через CDP
- Лиды с классифайдов → могут быть не с них, а с рекламы
- Как сегментировать аудиторию → например, для Программатика

### 2. Наличие динамики/статики

- Много обращений со статики → предложение Big Data
- Корректно ли используется статика → предлог для звонка
- Актуальность меток, актуальность номеров → предлог для звонка

### 3. Отслеживание заявок / прозвон форм

- Чем больше клиент завязан на наш продукт, тем лучше (Marquiz, чаты, докс)
- На заявки можно настроить CDP (заранее продумать 2-3 сценария)
- Премиум или Энтерпрайз — проверить ОЗ или прозвон форм
- Если много пропущенных звонков → предложить автоматизацию
- Дубли по заявкам → некорректная статистика
- При подключении SMS — с заявками проще исключить контакты из базы

### 4. Отчёт «Технологии и устройства»

- Сколько мобильных сессий → для предложения SMS или Войса
- Процент посещений с Safari больше 10% → срок жизни кук:
  > «(Имя клиента), обратила внимание, что у вас значимая доля сессий приходится на браузер Safari, а Safari ограничивает срок жизни кук до 7 дней. Так как история посещений — это фундамент для анализа эффективности рекламных источников, продление жизни кук с помощью Calltouch позволит вам получать корректные данные о посещениях независимо от того, насколько давно пользователь просматривал сайт.»

### 5. Корзина / электронная торговля

- Предложить настроить триггеры на товар, брошенный в корзину

### 6. Интеграция с CRM

- Всё ли с ней ок, все ли сделки связываются с лидами → предложить Big Data
- Сквозная аналитика → предлог помочь настроить дашборды
- Какая воронка/статусы → предложить CDP по зависшему статусу
- Дорогостоящие услуги → предложить CDP

### 7. Интеграция с РК

- Сколько денег тратят на рекламу
- Если бюджет больше 100к → предлагаем SMS, скоринг, программатик
- Если меньше 100к → Big Data, CDP в первую очередь
- Ключевые кампании → можно докрутить рассылками

### 8. Сфера деятельности

- Сезонность
- В несезон → докручиваем текущую базу
- Если сезон → привлечение новых
- Удачные кейсы по продуктам обезличенно

### 9. Специфика бизнеса

- Пеший трафик / обращения со статики / отслеживание офлайн по QR → Big Data
- Длительность пути до сделки:
  - Долгий — Скоринг, Войс, SMS
  - Быстрый — CDP
- Элитный бизнес или нет — не всегда готовы лишний раз пушить
- B2B или B2C — какие продукты будут актуальны
- Регион работы

### 10. Какая абонплата

- Сколько клиент тратит / готов тратить на сервис
- Энтерпрайз — многие продукты со скидкой

### 11. Какие продукты сейчас использует

- Какие могут быть актуальны, комплекс из нескольких продуктов, 2-3 сценария CDP
- Опыт работы с конкурентами, возможные боли

## Сценарии для разных бизнес-целей

### Если основная цель — **Сделки** (рост конверсии лида в сделку)

Вопросы:
- Какая воронка продаж?
- Где теряются больше всего сделок?
- Сколько в среднем дней от лида до сделки?
- Что делает менеджер для дожима?

Релевантные сценарии:
- Повторное посещение сайта → уведомление менеджеру
- Изменение статуса сделки → автоматический пинг

### Если основная цель — **Лиды** (рост конверсии в лид с сайта)

Вопросы:
- Какая текущая конверсия сайта?
- Откуда основной трафик?
- Какие услуги/продукты приоритетные?

Релевантные сценарии:
- Посещение страницы услуги → попап с УТП
- Долгое изучение сайта → виджет консультации
- Уход с сайта → экзит-попап

### Если основная цель — **Лояльность** (рост частоты повторных продаж)

Вопросы:
- Какой текущий retention?
- Через какое время клиенты возвращаются?
- Есть ли программа лояльности?

Релевантные сценарии:
- Реактивация по базе через файл
- Возвращение на сайт → персональное предложение
- Триггер по дате последней покупки

### Если основная цель — **Выручка** (рост среднего чека)

Вопросы:
- Какой средний чек сейчас?
- Какие маржинальные продукты?
- Есть ли cross-sell и up-sell?

Релевантные сценарии:
- Рекомендации сопутствующих товаров после покупки
- Триггер на дорогие категории → персональное обращение

### Если основная цель — **Автоматизация** (триггерные коммуникации)

Вопросы:
- Что сейчас делается вручную?
- Какие коммуникации повторяющиеся?
- Какие пинги забываете отправлять?

Релевантные сценарии:
- Welcome-цепочка после первого лида
- Триггер по календарю (день рождения, годовщина)
- Реактивация спящей базы

## Чек-лист "красные флаги" в брифе

Если клиент пишет в брифе так — задача требует уточнения:

🚩 **«Хочу больше лидов»** → Какой текущий объём? CR? Какая цель в цифрах?

🚩 **«Хочу попап»** → Какую задачу решает попап? Кому показывать?

🚩 **«У всех есть, и нам надо»** → Понимает ли клиент, для чего ему CDP?

🚩 **«Чтобы было как у конкурента X»** → Это шаблонное мышление, нужна индивидуальная стратегия

🚩 **«Хочу автоматизировать всё»** → Слишком абстрактно, нужны конкретные процессы

🚩 **Нет данных по среднему чеку** → Невозможно посчитать ROI

🚩 **Нет CRM** → Невозможно проследить путь до сделки

## Что должно быть в Дорожной карте по итогам выявления

После звонка-выявления Менеджер CDP формирует Дорожную карту с:

1. **Цели клиента** в цифрах (KPI)
2. **Топ 3 механики** под цели
3. **Метрики влияния** для каждой механики
4. **Сегмент аудитории** для каждого сценария
5. **Триггер запуска**
6. **Каналы коммуникации**
7. **Доп. работы при внедрении** (дизайн, интеграция, файл аудитории)
8. **Ожидаемый CPL** и расчёт окупаемости

## Открытие встречи (на повторных контактах)

> Хотим получить обратную связь по использованию нашего продукта Big data, Скоринг и т.д. Что можно улучшить в работе на нашей стороне?

> Обнаружили, что у вас на проекте большое количество сделок без связки с лидом. Она нужна, чтобы определить, насколько источники трафика эффективно влияют на конечные продажи. У нас есть 2 предложения, которые могут увеличить точность определения.

> Собираем обратную связь по сквозной аналитике — чего нам не хватает, чтобы заменить сквозную аналитику от Roistat?

> Проводим исследование, как клиенты используют продукт. Чего не хватает, что можно было бы улучшить?

> Хотим понять, какие проблемы со сквозной аналитикой у вас возникают? Что можем улучшить с точки зрения отчётов?

> Мы постоянно развиваемся, хотим послушать вашу историю работы с Calltouch. Возможно, есть какие-то настройки, которых вам не хватает.
