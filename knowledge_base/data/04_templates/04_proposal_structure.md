# 4. Структура индивидуального предложения

> Шаблон, по которому собирается коммерческое предложение CDP под конкретного клиента.

## Когда нужно индивидуальное предложение

- Клиент задал уточняющий вопрос про свой бизнес
- На стандартный буклет «не клюнул»
- Нужно дожать через персонализацию
- Высокий потенциальный чек (от 30к/мес абонки)

## Когда индивидуальное предложение НЕ нужно

- Клиент уже готов оплачивать (не загромождай продажу)
- Клиент попросил «просто прайс на почту»
- Клиент маленький с типовой задачей — хватит стандартного шаблона

---

## Структура предложения (8 блоков)

### Блок 1: Шапка

```markdown
# 🎯 Индивидуальное предложение CDP Calltouch

## Для [Название компании] ([URL сайта])
```

### Блок 2: Краткое резюме (3-5 предложений)

```markdown
### 📋 Краткое резюме

Подготовили для вас **[N] неформатных сценариев CDP**, заточенных под специфику [сферы клиента]: [ключевые особенности — цикл, чек, конкуренция].

**Главная цель:** [конкретная цель в одном предложении]

**Ожидаемый эффект:** [+%] к лидам в первый месяц при [текущих условиях клиента].

**Стоимость:** **10 500 ₽/мес** (при посещаемости до 60 000 визитов/мес).
```

### Блок 3: Понимание клиента

```markdown
## 🎯 Понимание клиента: что мы увидели на сайте

### Сильные стороны для CDP-механик

| Что есть | Как используем в CDP |
|----------|---------------------|
| [Конкретный факт о клиенте] | [Применение в сценарии] |
| ... | ... |

### Боли, которые решаем через CDP

1. **[Боль 1]** — [описание, почему это проблема]
2. **[Боль 2]** — ...
3. **[Боль 3]** — ...
```

> 💡 **Важно:** этот блок показывает клиенту, что мы изучили его бизнес, а не выслали шаблон. Используйте конкретику с сайта (имена специалистов, акции, услуги).

### Блок 4: 3-5 нестандартных сценариев

Для каждого сценария:

```markdown
### Сценарий №X: «[Название]» — [короткий смысл]

> **🎯 Цель:** [что решаем]
> **💰 Эффект:** [главная выгода]

#### Логика
[Объяснение на 2-3 предложения]

#### Карта сценария
```
[ASCII-схема потока]
```

#### Контент попапа (примерный)
> [Конкретный текст, который увидит клиент клиента]

#### Почему это нестандартно
- [Причина 1]
- [Причина 2]
- [Причина 3]

#### Метрики успеха
- [Целевые показатели]
```

> 💡 **Важно:** не больше 5 сценариев — клиент перегружен. Если очень хочется больше — выделите 2-3 «дополнительные идеи» в конце.

### Блок 5: Прогноз эффективности

```markdown
## 📊 Прогноз эффективности (консервативный)

При текущем трафике сайта (предположительно [N] тыс. визитов/мес) ожидаемые результаты:

| Метрика | Прогноз |
|---------|---------|
| Дополнительных лидов в месяц | [N] |
| Средний CPL через CDP | [N] ₽ |
| Конверсия в сделку | [N]% |
| Дополнительная выручка/мес | [N] ₽ |
| ROI по CDP | от [N]% |

> 💡 *Точный расчёт сделаем после получения данных по трафику и текущему среднему чеку.*
```

> ⚠️ **Не давай гарантий.** Используй формулировки «ожидаемый», «консервативный прогноз», «по нашим кейсам».

### Блок 6: Что потребуется для запуска

```markdown
## 🛠 Что потребуется для запуска

### От [Название компании]
1. **Заполнение брифа** (15-20 минут)
2. **Доступ к CRM** (уточним, что используется — мы интегрируем сами и **бесплатно**)
3. [Конкретные требования под сценарии: PDF, доступы, тексты]

### От Calltouch
1. ✅ Настройка всех [N] сценариев под ключ
2. ✅ Дизайн форм и попапов
3. ✅ Интеграция с CRM, сервисом email-рассылок
4. ✅ Настройка тегирования лидов и сделок
5. ✅ [Подключения по специфике клиента]
6. ✅ Отчёт эффективности через 7-30 дней с рекомендациями

### Сроки запуска
- Дорожная карта: **1-3 дня**
- Полный запуск всех сценариев: **8-12 рабочих дней**
```

### Блок 7: Дополнительные возможности (Phase 2)

```markdown
## 💎 Дополнительные возможности (фаза 2)

После того как базовые сценарии заработают и наберут статистику — рекомендуем подключить:

| Расширение | Что даёт | Цена |
|------------|---------|------|
| **CDP + Скоринг** | Сигналы об уходе к конкурентам | от +10 000 ₽/мес |
| **WABA** | Канал коммуникации с богатым медиа | от +30 000 ₽/мес |
| **Сквозная аналитика** | ROI до уровня сценария | по тарифу |
| **Big Data** | Оцифровка пешеходного трафика | по запросу |
```

> 💡 Этот блок — про будущее. Не давите, дайте понять, что есть куда расти.

### Блок 8: Призыв к действию (CTA)

```markdown
## 📞 Следующий шаг

Готовы заполнить бриф и подготовить **детальную дорожную карту** под [URL] с конкретными ID страниц, текстами попапов и расчётом маржинальности.

**Стоимость стартует с 10 500 ₽/мес.**
ROI по нашим кейсам в [отрасль] — **от [N]% до [N]%**.

В [релевантный кейс] аналогичный продукт принёс **+[N] [действия] и [N] ₽ выручки** — за счёт точечной коммуникации.

---

*Подготовлено командой Calltouch на основе анализа сайта [URL]*
```

## Подготовка предложения: пошаговый алгоритм

### Шаг 1: Изучить сайт клиента

Через web_fetch (или внутренний аналог) собрать:
- Категории услуг/товаров
- Действующие акции
- Имена руководителей и специалистов
- Уникальные предложения
- Сертификаты, награды, рейтинги
- Структуру страниц (для триггеров)

### Шаг 2: Определить отрасль

Использовать `03_clients/02_industry_specifics.md` для понимания:
- Особенности бизнеса
- Типовые задачи
- Релевантные кейсы

### Шаг 3: Проверить ограничения

Через `03_clients/03_restrictions.md`:
- Можно ли вообще предлагать CDP
- Какие категории под запретом
- Особенности отрасли (медицина — отдельная история)

### Шаг 4: Подобрать кейсы

Через `05_cases/` найти 1-2 релевантных кейса для прокачки доверия.

### Шаг 5: Сгенерировать сценарии

Минимум 3, максимум 5 сценариев с использованием:
- Базовых сценариев (`04_templates/01_basic_scenarios.md`)
- Расширенных (`04_templates/02_advanced_scenarios.md`)
- Креативных паттернов (`04_templates/03_creative_patterns.md`)

### Шаг 6: Просчитать ROI

Использовать формулу из `01_product/03_pricing.md`:
```
Минимальное число продаж = 10 500 / средний_чек × CR_лид_в_продажу
```

### Шаг 7: Собрать в один документ

Использовать структуру выше. Сохранить как .md в `/mnt/user-data/outputs/`.

## Чек-лист качества предложения

Перед отправкой клиенту проверьте:

- [ ] Использованы **конкретные имена** с сайта клиента
- [ ] Указана **реальная стоимость** услуг клиента
- [ ] Сценарии **используют реальные URL** с сайта клиента (или их паттерны)
- [ ] Есть **конкретные тексты попапов**, а не «попап с УТП»
- [ ] **3-5 сценариев**, не больше
- [ ] Каждый сценарий имеет **«почему это нестандартно»**
- [ ] Указан **прогноз ROI** с цифрами
- [ ] Использованы **релевантные кейсы** из той же отрасли
- [ ] **CTA** низкобарьерный («подготовлю бриф бесплатно»)
- [ ] Нет обещаний **гарантированного результата**
- [ ] Не упомянут **Smart SMS** (продажа приостановлена)
- [ ] Соблюдены **ограничения для медицины** (если применимо)

## Антипаттерны (чего избегать)

❌ **Шаблонность** — копипаст без учёта специфики клиента

❌ **Перегруз** — 10 сценариев лучше, чем 4? Нет, клиент устанет читать

❌ **Маркетинговый шум** — «революционная платформа», «уникальное решение» без конкретики

❌ **Точные гарантии** — «вы получите +147 лидов в месяц» (никто не может гарантировать)

❌ **Технические термины** — клиенту не нужно знать «webhook» и «триггер»

❌ **Сравнение с конкурентами** — без явного запроса

❌ **Длинные обоснования цены** — клиент сам поймёт, окупается ли

❌ **Игнорирование возражений** — если клиент говорил «дорого», покажи окупаемость
