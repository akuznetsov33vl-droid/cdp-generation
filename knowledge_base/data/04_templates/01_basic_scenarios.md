# 1. Базовые сценарии CDP Calltouch

> Эти 4 сценария — основа Дорожной карты для большинства клиентов. Они закрывают типовые задачи: рост конверсии в лиды и в сделки, работа с известными и неизвестными контактами.

## Сценарий 1: «Посещение сайта»

### Цель
Рост конверсии (CR в лид)

### Аудитория
Посетители сайта **без контактных данных** (новые)

### Триггер
Посещение сайта

### Канал
PopUp

### Логика

```
Аудитория: без контактов
         ↓
Начало сессии
         ↓
Показ PopUp с УТП-1 для новых клиентов
         ↓
Если сессия завершается без лида
         ↓
Ожидание: следующий заход на сайт
         ↓
Показ PopUp с другим УТП-2
```

### Описание для клиента

> Клиент, контакты которого мы не знаем, зашёл на сайт.
> Показываем баннер с УТП-1 для новых клиентов.
> Если сессия завершается без лида — ждём новый заход.
> Показываем ещё один баннер с другим УТП-2.
>
> Цепочку можно продлить, чтобы точечно воздействовать на заинтересованных, но сомневающихся посетителей.

### Доп. работы при внедрении
- Дизайн форм PopUp

## Сценарий 2: «Долгое изучение сайта»

### Цель
Рост конверсии (CR в лид)

### Аудитория
Посетители сайта **без контактных данных**

### Триггер
Долгое изучение (более 2 минут на сайте)

### Канал
PopUp / Промо-баннер с переходом в мессенджер

### Логика

```
Аудитория: без контактов
         ↓
Начало сессии
         ↓
Ожидание: 2 минуты на сайте
         ↓
Если лид не оставлен:
         ↓
Показ виджета с предложением консультации
ИЛИ
Показ Промо-баннера с кнопкой перехода в мессенджер
```

### Описание для клиента

> Посетитель без контактных данных зашёл на сайт. Провёл более 2 минут и не оставил лид.
> Показываем виджет с предложением консультации.
>
> Можно настроить показ Промо-баннера с кнопкой-перехода в мессенджер.
>
> **Преимущество перед обычной формой:** клиенту не нужно вводить контакты, он просто нажимает на кнопку и переходит в чат с менеджером.

### Доп. работы при внедрении
- Дизайн форм

## Сценарий 3: «Повторное посещение сайта»

### Цель
Рост конверсии в сделку (Лояльность / Сделки)

### Аудитория
Посетители сайта **с контактными данными** (известные клиенты из CRM или базы)

### Триггер
Повторное посещение сайта

### Канал
PopUp / Сообщение клиенту / Email менеджеру

### Логика

```
Аудитория: с контактами (повторные)
         ↓
Начало сессии
         ↓
Показ баннера с предложением «для своих»
         ↓
Ожидание завершения сессии
         ↓
Если лид не поступил:
         ↓
Сообщение клиенту (email/SMS)
ИЛИ
Email менеджеру: «клиент вернулся, звоните»
```

### Описание для клиента

> Клиент с известными контактными данными возвращается на сайт.
> Показываем баннер с предложением «для своих».
> Ждём завершение сессии.
> Если лид так и не поступил — направляем сообщение клиенту или email менеджеру.
>
> В аудиторию сценария можно загрузить контакты клиентов файлом xls из существующей базы.

### Доп. работы при внедрении
- Дизайн формы
- Интеграция с сервисом рассылок
- Загрузить аудиторию в файл

## Сценарий 4: «Новый лид»

### Цель
Рост конверсии в сделку (Лояльность / Сделки)

### Аудитория
Посетители сайта **с контактными данными** (только что оставили лид)

### Триггер
Новый лид

### Канал
Сообщение клиенту (welcome-цепочка)

### Логика

```
Аудитория: с контактами
         ↓
Триггер: Новый лид
         ↓
Приветственное сообщение (SMS/Email)
         ↓
Ожидание: 3 дня
         ↓
Email с информацией о компании + актуальные акции
         ↓
Ожидание: 7 дней
         ↓
Email с дополнительной мотивацией к покупке
```

### Описание для клиента

> Посетитель сайта оставил новый лид.
> Направляем приветственное сообщение клиенту.
>
> Далее можно настроить отправку серии писем (например, email) с информацией о клинике/компании, описанием актуальных акций и т.д.

### Доп. работы при внедрении
- Интеграция с сервисом рассылок

## Когда использовать какой сценарий

### Если у клиента низкая конверсия сайта (< 2%)
**Сценарии 1 + 2** — поведение посетителей без контактов

### Если у клиента есть база, но мало повторных продаж
**Сценарий 3** — реактивация и работа с возвращающимися

### Если у клиента есть лиды, но много теряется в воронке
**Сценарий 4** — welcome-цепочка для дожима

### Если все три проблемы сразу
**Полный пакет: 1 + 2 + 3 + 4** — комплексное закрытие воронки

## Стандартное предложение в Дорожной карте

Для большинства клиентов в Дорожной карте предлагают **топ-3 механики** (а не все 4) под основную цель клиента.

**Обязательно предлагается лидогенерация** (сценарии 1 или 2), если у клиента есть Обратный звонок:
- У проекта обратный звонок → **сценарий с показом форм ОЗ**
- У проекта нет обратного звонка → **сценарий с показом форм УЗ** (умной заявки)
