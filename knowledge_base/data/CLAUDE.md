## 🎯 Как использовать эту базу знаний

### Сценарий 1: Подготовка индивидуального предложения для нового клиента

**Контекст:** Клиент даёт ссылку на свой сайт, нужно подготовить предложение.

**Алгоритм:**
1. Изучить сайт клиента (web_fetch)
2. Определить отрасль → проверить раздел `03_clients/02_industry_specifics.md`
3. **Найти готовые механики для отрасли** → `04_templates/05_industry_communication_maps.md` ⭐
4. Найти релевантные кейсы → раздел `05_cases/`
5. Проверить ограничения → `03_clients/03_restrictions.md` (особенно для медицины)
6. Составить 3-5 нестандартных сценариев, **комбинируя готовые механики из карты + креативные паттерны** → `04_templates/02_advanced_scenarios.md` + `04_templates/03_creative_patterns.md`
7. Использовать структуру предложения → `04_templates/04_proposal_structure.md`
8. Добавить расчёт ROI → `01_product/03_pricing.md`

### Сценарий 2: Ответ на возражение клиента

**Контекст:** Клиент сомневается, сравнивает с альтернативами или не понимает ценности.

**Алгоритм:**
1. Идентифицировать тип возражения → `02_sales/02_objections.md`
2. Если возражение из **свежей фокус-группы** (сентябрь 2025): «приоритет на новых клиентов», «есть бесплатная автоматизация в CRM», «некому обрабатывать сигналы», «маленькая база», «хотят гарантий», «лидген за 500₽ дешевле» → `02_sales/08_webinar_september_2025_new_process.md` ⭐ (расширенная база возражений)
3. Если возражение про Тильду/попапы → `02_sales/05_cdp_vs_tilda.md`
4. Если возражение про **конкретного конкурента** (Mindbox, Altcraft, Carrotquest, Sendsay, Roistat, enKod, Rees46, edna) → `02_sales/06_positioning_webinar.md` ⭐ — там детальное сравнение по 10 параметрам
5. Если возражение про CRM-роботов / ручной труд / инхаус → `02_sales/06_positioning_webinar.md` ⭐ (раздел «вторичные конкуренты»)
6. Если возражение касается **позиционирования** или «зачем продукт» → `02_sales/07_webinar_august_2025.md` ⭐ (глубокая аргументация от продакта)
7. Найти релевантный кейс из той же отрасли → `05_cases/`

### Сценарий 3: Подготовка встречи с клиентом по новой методологии (с ноября 2025)

**Контекст:** Менеджер по продажам идёт на встречу с клиентом и должен «продать идею».

**Алгоритм:**
1. Изучить методологию ВСПВ → `02_sales/08_webinar_september_2025_new_process.md` ⭐
2. Подготовиться к выявлению потребностей → `02_sales/04_discovery_questions.md`
3. Подготовить релевантные кейсы → `05_cases/`
4. Освежить ключевые цифры эффективности (1₽ → 36₽ прибыли, до 25 раз дороже привлекать новых, до 95% рост конверсии) → `02_sales/08_webinar_september_2025_new_process.md`
5. **Проверить калькулятор маржинальности** на сайте Calltouch (открыть на встрече)
6. Если возможно — заранее запросить медиаплан в ОБА

### Сценарий 4: Разработка нестандартных сценариев CDP

**Контекст:** Стандартные 4 сценария уже предложены, нужно что-то креативное.

**Алгоритм:**
1. Понять цели клиента → `02_sales/04_discovery_questions.md`
2. Использовать паттерны → `04_templates/03_creative_patterns.md`
3. Проверить технические возможности → `01_product/02_architecture.md` (раздел триггеры/действия)
4. Проверить ограничения → `01_product/05_technical_nuances.md`

### Сценарий 5: Расчёт стоимости и окупаемости

**Контекст:** Клиент спрашивает «сколько это стоит» и «когда окупится».

**Алгоритм:**
1. Тарификация → `01_product/03_pricing.md`
2. **Калькулятор маржинальности на сайте Calltouch** → инструкция в `02_sales/08_webinar_september_2025_new_process.md`
3. Релевантные кейсы с ROI → `05_cases/`

## ⚙️ Ключевые принципы работы AI-ассистента

### При составлении предложений

1. **Всегда изучай сайт клиента** перед предложением — конкретика побеждает шаблоны
2. **Используй имена с сайта клиента** (тренеров, врачей, услуг) — это показывает погружение
3. **Не предлагай больше 5 сценариев** — клиент перегружен
4. **Каждый сценарий должен иметь объяснение «почему это нестандартно»** — это отличает от шаблонов
5. **Считай ROI на конкретных цифрах клиента**, а не в общих процентах

### При отработке возражений

1. **Соглашайся с клиентом сначала** — снимает оборону («вы правы, что...»)
2. **Никогда не критикуй текущие решения клиента** — они работают, иначе клиент бы не выжил
3. **Переворачивай возражения в аргументы** (классика: «мало трафика» → «значит, дорогой каждый клик»)
4. **Конкретные примеры всегда сильнее абстракции**
5. **Заканчивай низким барьером входа** — «бесплатно подготовлю», «давайте посмотрим»

### Ключевые ограничения

1. **Медицина** — специфические запреты на формулировки (нельзя предполагать болезнь)
2. **Категории под запретом** — лекарства по рецепту, оккультные услуги, политика, табак и др.
3. **Минимальные пороги** — для ряда расширений (Скоринг, WABA, Smart SMS)
4. **Smart SMS Билайн** — продажа ПРИОСТАНОВЛЕНА (важно не обещать)
5. **WhatsApp** — может быть запрещён в РФ; серый WhatsApp Calltouch не делает (только белый WABA)
6. **Email как канал** — технически готов, юридические блокеры; ожидаемый запуск: октябрь 2025 — январь 2026
7. **Smart SMS МТС/Мегафон** — никогда не будет (МТС нет технологии, Мегафон не хочет)

### Новый процесс продажи (с ноября-декабря 2025)

1. **Менеджер по продажам продаёт ИДЕЮ** — не детали и не сценарии
2. **После оплаты счёта** — клиент передаётся в **Отдел бизнес-аналитики (ОБА)**
3. **ОБА готовит медиаплан** с конкретными сценариями для клиента
4. **ОБА настраивает** триггеры, делает срез эффективности
5. На встрече **используем калькулятор маржинальности** на сайте Calltouch для конкретики
