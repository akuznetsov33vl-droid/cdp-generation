# 2. Архитектура платформы CDP Calltouch

## Последовательная структура CDP

```
Источники данных → Профиль клиента → Сегментация → Триггеры → Коммуникация
```

Каждый клиент проходит через все 5 блоков, и на каждом — обогащается.

## Блок 1: Источники данных

| Источник | Что даёт |
|----------|----------|
| **CRM** | Сделки, статусы, история покупок |
| **Web** | Поведение на сайте, страницы, время |
| **АТС** | Звонки, длительность, исход |
| **Calltracking** | Источник трафика для каждого звонка |
| **Внешние данные (Big Data)** | Интерес клиента к отрасли вне сайта |

## Блок 2: Профиль клиента (360° view)

Шесть категорий данных в едином профиле:
- **Демография** — возраст, пол, локация
- **Поведение** — страницы, время, частота визитов
- **Транзакции** — покупки, чек, повторные продажи
- **Коммуникации** — каналы и история взаимодействий
- **Атрибуты** — кастомные параметры
- **Скоринг** — рассчитанный балл заинтересованности

## Блок 3: Сегментация

| Тип сегмента | Описание | Когда использовать |
|--------------|----------|-------------------|
| **Гибкая сегментация** | По любым атрибутам, 100+ параметров | Сложные кастомные сценарии |
| **Динамические аудитории** | Автообновление в реальном времени | Триггерные сценарии, реактивация |
| **Статичные сегменты** | Фиксированные группы клиентов | Разовые рассылки, кампании |

### Параметры сегментации (примеры)

- Демографические данные
- Поведение на сайте (страницы, время, источники)
- История лидов (звонки, заявки, чаты)
- Сделки и транзакции
- Контактные данные (есть/нет, тип)
- Источники трафика и UTM-метки
- Технологии (браузер, устройство)
- География
- Скоринг и заинтересованность
- Кастомные атрибуты

### Источники аудиторий

1. **Из проекта** — собранные на основе аналитики Calltouch
2. **Из файла** — импорт клиентской базы (xls)
3. **Зашифрованная база** — формат sha256 (через менеджеров CDP)

### Временной срез касаний

| Значение | Поведение |
|----------|-----------|
| **Любой период касаний** | Все клиенты независимо от давности |
| **Не задано (по умолчанию)** | Касание от 1 дня |
| **Указанный период** | Только клиенты, касавшиеся в диапазон |

> **Касания** = заявки, звонки, сессии, сделки

## Блок 4: Триггеры и события

### Полный список доступных триггеров

| Триггер | Описание |
|---------|----------|
| **Начало сессии** | Клиент зашёл на сайт |
| **Завершение сессии** | Клиент вышел с сайта |
| **Новый лид** | Поступила заявка/звонок |
| **Изменение лида** | Изменились параметры лида |
| **Новая сделка** | Появилась сделка в CRM |
| **Изменение статуса сделки** | Сделка перешла в новый статус |
| **Просмотр страницы** | Открыта определённая страница |
| **Достижение цели** | JS-цель сработала (добавление в корзину и т.п.) |
| **Выявлен интерес** | Скоринг определил заинтересованность (требует Скоринг) |
| **Пользовательское событие** | Кастомное событие через API |
| **По расписанию** | Запуск по таймеру |

### Категории триггеров

| Категория | Примеры |
|-----------|---------|
| **Поведенческие** | Посещение сайта, брошенная корзина, покупка |
| **Временные** | День рождения, годовщина покупки |
| **Скоринговые** | Интерес в отрасли |

### Ожидание событий

Ожидание — это период, в течение которого ждём действие клиента. **Не может использоваться первым в сценарии.**

Доступные ожидания:
- Ожидание начала сессии
- Ожидание завершения сессии
- Ожидание лида
- Ожидание изменения лида
- Ожидание сделки
- Ожидание изменения статуса сделки
- Ожидание изменения цели
- Ожидание просмотра страницы
- Ожидание пользовательского события

### Логика срабатывания (важно!)

> Сценарий запускается и проверяет наличие клиента в аудитории **в момент срабатывания триггера**. Сначала идёт СОБЫТИЕ, затем проверяется аудитория.

**Пример:** Аудитория → Завершение сессии → Действие
1. Когда сессия завершится, проверим, попадает ли клиент под аудиторию
2. Если подойдёт — запустим сценарий

## Блок 5: Действия и каналы коммуникаций

### Типы действий в сценариях

| Действие | Описание |
|----------|----------|
| **Email менеджеру** | Уведомление сотруднику |
| **Email клиенту** | Письмо клиенту через шлюз Calltouch |
| **SMS менеджеру** | SMS сотруднику |
| **SMS обращавшимся клиентам** | SMS клиенту с известным номером |
| **SMS потенциальным клиентам** (Smart SMS) | Только в разовых рассылках; **продажа приостановлена** |
| **Обратный звонок** | Колбек через виджет |
| **Webhook** | Отправка данных во внешние системы |
| **Показ формы** | PopUp с формой виджета |
| **Тегирование клиента** | Присвоение тега для аналитики |
| **Сообщение в мессенджер** | WhatsApp, Max, VK, Telegram |

### Поддерживаемые мессенджеры

| Мессенджер | Способ интеграции |
|------------|-------------------|
| WhatsApp (WABA) | TextBack, WazzUp24, SigmaSMS |
| WhatsApp прочее | Через Albato |
| Max | WazzUp24 |
| VK | Прямая интеграция |
| Telegram | Через Albato (бот) |
| Viber | Через сервисы рассылок |

### Особенности отправки в Max

- Отправка возможна только по номеру телефона пользователя Max
- Диалог должен быть начат пользователем ранее
- «На пустом месте» отправлять нельзя (политика Max)
