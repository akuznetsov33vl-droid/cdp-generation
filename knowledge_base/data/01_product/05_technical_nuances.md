# 5. Технические нюансы, ограничения, FAQ

## Общие нюансы сценариев

| Нюанс | Описание |
|-------|----------|
| **1 номер + 1 почта на клиента** | В сценарий запускается клиент максимум с 1 номером и/или 1 почтой; ограничений по cookies нет |
| **Календарные дни** | Касание от 1 дня = клиент попадает в сценарий после 00:00 этого дня |
| **Один клиент — одна цепочка** | Клиент не попадёт в сценарий повторно, пока предыдущая цепочка не завершилась |
| **Сначала событие, потом аудитория** | Сценарий проверяет аудиторию в момент срабатывания триггера |
| **Лид во время сессии** | Параметр «Отсутствие лида в рамках сессии» учитывает все лиды за время сессии (даже после ухода с сайта) |
| **Ретростата 7 дней** | Может не показываться, если в сценарий не вносили изменений более 7 дней или услуга «Приостановлена» |

## Ограничения по запуску сценариев

Запуск **невозможен**, если:
1. Статус «Демо-режим» или «Приостановлен»
2. Дата и время рассылки меньше текущих
3. Недостаточно средств на счёте для рассылки SMS
4. Прогнозируемый расчёт количества действий = 0
5. Виджет обратного звонка деактивирован
6. На карте есть несвязанный элемент
7. В элементе не выбраны все настройки
8. Аудитория с исключёнными номерами + действие «Обратный звонок»
9. По рассылке ожидается меньше **500 отправок**

## Важные нюансы при настройке аудитории

### Триггер «Завершение сессии: с лидом»

При сценарии `Аудитория (с контактами) → Завершение сессии: с лидом → Действие`:
- Клиент без контактов оставил лид во время сессии → к концу сессии у него уже есть контакты → попадёт в сценарий

**Как исправить (только повторные клиенты):**
```
Аудитория → Начало сессии → Ожидание завершения сессии (с лидом) → Действие
```

### Триггер «Новый лид»

Сценарий `Аудитория (с контактами) → Новый лид → Действие`:
- Клиент оставил лид → у него появились контакты одновременно → сценарий запустится

**Как исправить:**
```
Аудитория → Новый лид (повторный) → Действие
```

### Триггер «Новая сделка»

Если ожидаем сделку от клиентов с уже существующей сделкой — фильтруем аудиторию по наличию сделок.

## FAQ по функциональности

### ❓ Можем ли отправлять сообщения в Max?

Да, но по номеру телефона пользователя Max — диалог должен быть начат ранее. Через wazzup24.ru.

### ❓ Можем ли в WhatsApp/VK/Telegram?

Да:
- WhatsApp — TextBack, WazzUp24, SigmaSMS
- Max — WazzUp24
- VK — прямая интеграция
- Telegram — через Albato (бот)

### ❓ Можем ли настроить брошенную корзину?

Да. Через JS-цель на добавление товара + сценарий с триггером «Достижение цели» и ожиданием сделки.

```
Аудитория → Триггер "Достижение цели" → Ожидание сделки (N минут): не наступило → Показ формы / SMS / Webhook
```

### ❓ Можем ли реактивировать клиентов, не обращавшихся 60+ дней?

Да:
```
Аудитория → Новый Лид → Ожидание Лида (60 дней) → Не наступило → Действие
```

### ❓ Можем ли отслеживать заявку с конкретной формы?

Да, через тегирование форм и фильтр по тегу в триггере «Новый лид».

### ❓ Можем ли отправлять письма на почту клиенту?

Да, тремя способами:
1. **E-mail клиенту** (Шлюз Колтач)
2. **Webhook** (Шлюз Unisender / Notisend)
3. **Кастомная интеграция** со сторонним сервисом

### ❓ Можем ли работать с зашифрованной базой?

Да, формат **sha256**. За подробностями — к Ричарду Мартинесу или Кириллу Шейкину.

### ❓ Можем ли реагировать на бездействие (более 30 секунд без заявки)?

Да:
```
Начало сессии → Ожидание лида (нужный интервал) → не наступило → Действие
```

### ❓ Можем ли отслеживать брошенную форму на половине заполнения?

Да, через условия в «Завершении сессии»:
- `Цели = заполнил шаг 2 И Цели ≠ заполнил шаг 3`

### ❓ Можем ли передавать «Товары в корзине» в коммуникацию?

Да, через макросы `{товары_в_корзине}` и `{заказанные_товары}` в настройках письма/вебхука.

### ❓ Можем ли отправлять цепочку касаний?

Да:
```
Триггер → Действие 1 → Ожидание (N дней) → Действие 2 → Ожидание → Действие 3
```

### ❓ Что делать, если клиент жалуется, что попап показывается ему повторно?

В сценарий встроена логика: один клиент = одна активная цепочка. Если клиент уже прошёл через сценарий — повторно не попадёт, пока цепочка не завершится.

## FAQ по работе сценария

### ❓ Если клиент уже в сценарии, попадёт ли он повторно?

Нет, пока предыдущая цепочка не завершилась.

**Пример:**
- 1 касание: 19 ноября — клиент оставил лид и перешёл на этап ожидания сделки (1 день)
- 2 касание: 19 ноября — клиент оставил повторный лид. **В рамках этого же сценария новую цепочку не запустим**, потому что он ещё не вышел из блока «Ожидания»

### ❓ Когда клиент попадает в сценарий с касанием от 1 дня?

После **00:00** этого дня (учитываются календарные дни).

### ❓ Почему ретростата может не отображаться?

Если в сценарий не вносили изменений более 7 дней или услуга в статусе «Приостановлен».

### ❓ Что значит «отрицания через ИЛИ»?

Если отрицания объединены через ИЛИ, это равнозначно отсутствию каких-либо условий.

**Пример:** НЕ "дверь" ИЛИ НЕ "окно".
Проверяю окно:
- Окно не дверь? Да, это не дверь
- Достаточно выполнения одного условия? Да, потому что выбрано ИЛИ
- **Итог:** окно пройдёт фильтр

## FAQ по стоимости

### ❓ Как считается абонплата при нескольких сайтах?

- Один siteID = одна а/п
- Все сайты на одном приложении → общая а/п
- Разные приложения → отдельные а/п (нужно следить за корректностью в админке)

### ❓ Что входит в минимальную цену 10 500 ₽?

- Триггерные коммуникации
- Рассылка вебхуков
- Бесплатные интеграции
- Настройка под ключ
- Первый отчёт эффективности

## Поведенческие особенности

### Если в сценарии используется триггер «Новый лид» и поступает звонок

Триггер срабатывает по факту **начала звонка**.

**Пример:**
```
Аудитория → Новый лид → Ожидание начала сессии → Действие
```

Клиент позвонил в компанию, идёт разговор. За время разговора клиент параллельно зашёл на сайт (его номер и кука связаны) → сценарий отработает, так как триггер «Новый лид» уже сработал.

## Тестирование сценариев

### Через тестовую ссылку

```
site.ru?ct_scenario_test=контакт
```

где:
- `site.ru` — сайт клиента
- `контакт` — номер (`79209366632`) или почта (`test@test.ru`)

**Что произойдёт:**
1. Клиент попадает в сценарий на первый шаг (если он сессионный)
2. Выполняйте действия согласно сценарию (для попадания в условия триггера)

> ⚠️ Тип коммуникации не зависит от того, что вы указали в тестовой ссылке. Если по сценарию проверяете обратный звонок или SMS — указывайте заведомо рабочий номер. **Деньги за СМС спишутся.**

## Кастомная аналитика и теги

### Тегирование лидов

При срабатывании сценария на лид/сделку клиента ставится тег:
- **«Клиент проходил через триггерный сценарий»** — задан по умолчанию

Кастомные теги имеет смысл добавлять только если нужно разделять статистику по разным триггерам.

### Окно ретроспективы

В настройках сценария указывается количество дней, в пределах которого лид/сделка получат источники от сценария. По умолчанию — **3 дня**.

**Пример:**
- Вася вернулся на сайт → отправили менеджеру письмо-уведомление
- Гена весь вечер уговаривал клиента
- На следующий день Вася покупает пеноблоки
- Этой сделке мы дадим тег «Клиент проходил через триггерный сценарий»

## Уведомления для менеджеров клиента

При работе сценариев Calltouch отправляет автоматические письма:

| Шаблон | Когда |
|--------|-------|
| `calltouch-leads-scenarios-deactivated` | Сценарий деактивирован |
| `calltouch-leads-zero-balance` | Баланс на нуле |
| `calltouch-leads-notif-no-triggers-after-10-days` | Нет срабатываний 10 дней |
| `calltouch-leads-notif-create-audience-and-no-scenario` | Создана аудитория, но нет сценария 5 дней |
| `calltouch-leads-notif-upsale-trigger-client-sessions` | Превышен тариф по сессиям |
| `calltouch-email_gateway-change-status-of-domain` | Изменён статус домена для email |

## Особенности WABA

На момент перехода к подключению WABA Calltouch к CDP нужно иметь:
1. Оплаченный счёт по WABA
2. Созданный аккаунт в Textback
3. Зарегистрированный номер в аккаунте Textback
